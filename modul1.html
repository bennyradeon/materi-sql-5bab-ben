Modul 1 — SQL Dasar (DDL & DML)

Di tahap awal belajar, kalian perlu mengenali perintah-perintah SQL yang paling sering dipakai. Modul ini memperkenalkan DDL (mengatur struktur tabel) dan DML (mengolah isi data).

Kamus Istilah (Modul 1)

Bagian ini sengaja saya tulis lengkap karena istilah seperti ini sering muncul di soal.

  • SQL = Structured Query Language → bahasa untuk mengakses dan mengelola data di database relasional.
  • DBMS = Database Management System → perangkat lunak pengelola database (contoh: MySQL, MariaDB).
  • RDBMS = Relational DBMS → DBMS berbasis tabel yang saling berelasi (MySQL termasuk RDBMS).
  • DDL = Data Definition Language → perintah untuk struktur (CREATE/ALTER/DROP).
  • DML = Data Manipulation Language → perintah untuk isi data (INSERT/UPDATE/DELETE/SELECT).
  • PK = Primary Key → kolom identitas unik tiap baris.
  • FK = Foreign Key → kolom penghubung ke PK tabel lain (membentuk relasi).
  • AI = Auto Increment → penomoran otomatis pada kolom INT (umumnya untuk id).
Tips: Saat ujian, biasakan menulis kepanjangan istilah dulu, baru fungsinya, baru contoh perintahnya.
Catatan penting sebelum praktik

Saat kalian menulis SQL, biasakan memeriksa tiga hal: nama tabel, nama kolom, dan tipe data. Kesalahan paling sering adalah salah eja kolom atau salah format nilai (misalnya tanggal).

  • INT untuk angka bulat (misal id, stok).
  • VARCHAR untuk teks pendek (misal nama, email).
  • DATE untuk tanggal (format: YYYY-MM-DD).
  • PRIMARY KEY untuk identitas unik setiap baris.
  • FOREIGN KEY untuk menghubungkan tabel (relasi).
Materi yang sering muncul
  • Perbedaan DDL dan DML + contoh perintahnya.
  • CREATE/DROP DATABASE, CREATE/ALTER/DROP TABLE.
  • UPDATE/DELETE aman (wajib WHERE).
  • Tipe data: INT, VARCHAR, DATE, DATETIME, FLOAT.
  • PRIMARY KEY dan FOREIGN KEY.
1) DDL vs DML

DDL adalah singkatan dari Data Definition Language, yaitu kumpulan perintah untuk membuat, mengubah, dan menghapus struktur database atau tabel. Contoh perintah DDL: CREATE DATABASE, CREATE TABLE, ALTER TABLE, DROP TABLE.

DML adalah singkatan dari Data Manipulation Language, yaitu kumpulan perintah untuk mengelola isi data di dalam tabel (menambah, mengubah, menghapus, menampilkan data). Contoh perintah DML: INSERT, UPDATE, DELETE, SELECT.

Supaya kalian tidak tertukar, ingat ini: DDL mengatur wadahnya (tabel/kolom), sedangkan DML mengatur isi di dalam wadah (baris data).

  • DDL: “membangun rak & label di lemari arsip”
  • DML: “menaruh, memperbaiki, menghapus, dan mencari berkas”
Contoh nyata DDL vs DML:
• DDL: "Saya membuat tabel siswa dengan kolom nis, nama, alamat."
• DML: "Saya menambahkan data siswa baru, lalu menampilkan daftar siswa."
2) Wajib pakai WHERE

Saat UPDATE atau DELETE, kalian wajib hati-hati. Tanpa WHERE, data bisa ikut berubah/hilang semuanya.

Contoh Praktik Cepat

Jalankan di phpMyAdmin (tab SQL). Silakan sesuaikan nama tabel/kolom.

A) CREATE TABLE
-- DDL (Data Definition Language): membuat struktur tabel baru
CREATE TABLE ekskul (
  id_ekskul INT AUTO_INCREMENT PRIMARY KEY,
  nama_ekskul VARCHAR(50) NOT NULL,
  pembina VARCHAR(100)
);

-- Contoh DDL lain (mengubah struktur): menambah kolom ke tabel siswa
ALTER TABLE siswa
ADD COLUMN email VARCHAR(100);
B) INSERT, UPDATE, DELETE
-- DML (Data Manipulation Language): mengelola isi data

-- INSERT: menambah data ekskul
INSERT INTO ekskul (nama_ekskul, pembina)
VALUES ('Pramuka', 'Pak Rahmat');

-- UPDATE: mengubah data (ingat: wajib WHERE)
UPDATE siswa
SET alamat = 'Jakarta Selatan'
WHERE nis = 'S001';

-- DELETE: menghapus data (ingat: wajib WHERE)
DELETE FROM peminjaman
WHERE id_pinjam = 2;
C) SELECT
-- SELECT: menampilkan data

-- 1) cek isi tabel
SELECT * FROM siswa;

-- 2) tampilkan kolom tertentu (lebih rapi)
SELECT nis, nama, alamat
FROM siswa
ORDER BY nama ASC;

-- 3) filter data (WHERE)
SELECT nis, nama, tahun_masuk
FROM siswa
WHERE tahun_masuk = 2023;

Instruksi pengerjaan
  1. Pilih database: jalankan USE db_sekolah; (atau pilih dari phpMyAdmin).
  2. Tulis query dulu di catatan, lalu jalankan di phpMyAdmin (tab SQL).
  3. Pastikan benar: cek nama kolom, tanda petik, dan hasil output.
  4. Simpan bukti: screenshot query + tabel hasilnya.
  5. Kumpulkan: file PDF/Doc berisi jawaban (nomor + query + screenshot).
Soal latihan
Latihan 1 — DDL (struktur)
  1. Buat tabel ekstrakurikuler (id INT PK AI, nama VARCHAR(50)).
  2. Tambahkan 3 data ekskul.
  3. Tampilkan semua data ekskul.
Latihan 2 — DML (isi data)
  1. Update salah satu nama ekskul (wajib pakai WHERE).
  2. Delete 1 data ekskul (wajib pakai WHERE).
  3. Tuliskan 1 kalimat: apa risiko UPDATE/DELETE tanpa WHERE?
Latihan 3 — SELECT + filter
  1. Tampilkan siswa yang tahun_masuk = 2023 (kolom: nis, nama, alamat).
  2. Tampilkan siswa perempuan yang alamatnya Jakarta.
  3. Urutkan hasil berdasarkan nama A–Z.
Catatan guru: Jika hasil kosong, cek lagi syaratnya. Bisa jadi datanya memang tidak memenuhi kondisi. Kalian boleh menyesuaikan angka batas (misal 80, 85) asal logikanya tetap benar.
