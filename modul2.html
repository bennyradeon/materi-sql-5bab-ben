Modul 2 — Fungsi Agregat + GROUP BY

Di modul ini kalian belajar membuat “laporan ringkas” dari sekumpulan data. Fungsi agregat mengubah banyak baris menjadi satu nilai (jumlah, total, rata-rata, min, max).

Kamus Istilah (Modul 2)

Kalau kalian paham istilah ini, soal agregat biasanya jadi mudah.

  • AGGREGATE = fungsi ringkasan → mengubah banyak baris menjadi satu nilai (COUNT/SUM/AVG/MIN/MAX).
  • COUNT = menghitung jumlah baris.
  • SUM = menjumlahkan total.
  • AVG = menghitung rata-rata.
  • DISTINCT = mengambil nilai unik (tanpa duplikat).
  • GROUP BY = mengelompokkan baris sebelum dihitung agregat.
  • HAVING = menyaring hasil agregat setelah GROUP BY.
  • WHERE = menyaring baris sebelum GROUP BY.
Cara berpikir agregat (biar tidak bingung)

Agregat dipakai saat kalian tidak lagi ingin melihat daftar detail, tetapi ingin melihat ringkasan. Misalnya: jumlah siswa, total SKS, rata-rata nilai. Jika kalian memakai agregat, biasanya kalian akan bertemu GROUP BY.

  • GROUP BY = mengelompokkan baris menjadi beberapa kelompok.
  • COUNT/SUM/AVG = menghitung ringkasan di tiap kelompok.
  • HAVING = menyaring hasil ringkasan (bukan menyaring baris mentah).
Materi yang sering muncul
  • COUNT, SUM, AVG, MIN, MAX dan kapan dipakai.
  • DISTINCT untuk nilai unik.
  • GROUP BY untuk rekap per kelompok.
  • Perbedaan WHERE dan HAVING.
1) Fungsi Agregat
  • COUNT() menghitung jumlah data
  • SUM() menjumlahkan total
  • AVG() menghitung rata-rata
  • MIN() nilai terkecil
  • MAX() nilai terbesar
2) DISTINCT, GROUP BY, HAVING

DISTINCT menghapus duplikasi. GROUP BY mengelompokkan. HAVING menyaring hasil agregat (karena WHERE tidak boleh berisi agregat).

Ingat: WHERE sebelum GROUP BY, HAVING setelah GROUP BY.
Contoh Praktik

Contoh memakai tabel matakuliah (kode_mk, nama_mk, sks, semester).

A) DISTINCT
-- DISTINCT: ambil nilai unik (tanpa duplikat)
SELECT DISTINCT alamat
FROM siswa
ORDER BY alamat;
B) Agregat dasar
-- Fungsi agregat (ringkasan data)

-- COUNT: jumlah siswa
SELECT COUNT(*) AS jumlah_siswa
FROM siswa;

-- AVG: rata-rata nilai (semester 1)
SELECT AVG(nilai_angka) AS rata_rata_nilai
FROM nilai
WHERE semester = 1;

-- MIN & MAX: nilai terendah dan tertinggi
SELECT MIN(nilai_angka) AS nilai_min, MAX(nilai_angka) AS nilai_max
FROM nilai;

-- SUM: total stok buku
SELECT SUM(stok) AS total_stok_buku
FROM buku;
C) GROUP BY + HAVING
-- GROUP BY: rekap per kelompok

-- Rekap jumlah siswa per alamat
SELECT alamat, COUNT(*) AS jumlah_siswa
FROM siswa
GROUP BY alamat
ORDER BY jumlah_siswa DESC;

-- Rekap jumlah peminjaman per status, tampilkan hanya yang jumlahnya >= 2
SELECT status, COUNT(*) AS jumlah
FROM peminjaman
GROUP BY status
HAVING COUNT(*) >= 2;

Instruksi pengerjaan
  1. Pilih database: jalankan USE db_sekolah; (atau pilih dari phpMyAdmin).
  2. Tulis query dulu di catatan, lalu jalankan di phpMyAdmin (tab SQL).
  3. Pastikan benar: cek nama kolom, tanda petik, dan hasil output.
  4. Simpan bukti: screenshot query + tabel hasilnya.
  5. Kumpulkan: file PDF/Doc berisi jawaban (nomor + query + screenshot).
Soal latihan
Latihan 1 — COUNT & DISTINCT
  1. Hitung jumlah siswa (COUNT).
  2. Hitung jumlah alamat unik siswa (COUNT DISTINCT alamat).
Latihan 2 — GROUP BY
  1. Kelompokkan siswa berdasarkan alamat, hitung jumlahnya.
  2. Kelompokkan nilai berdasarkan mapel, hitung rata-rata nilainya.
Latihan 3 — HAVING
  1. Tampilkan alamat yang jumlah siswanya ≥ 2 (pakai HAVING).
  2. Tampilkan mapel yang rata-rata nilainya ≥ 85 (pakai HAVING).
Catatan guru: Jika hasil kosong, cek lagi syaratnya. Bisa jadi datanya memang tidak memenuhi kondisi. Kalian boleh menyesuaikan angka batas (misal 80, 85) asal logikanya tetap benar.
