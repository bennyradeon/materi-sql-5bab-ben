Modul 3 — Relasi, Join, dan Union

Di modul ini kalian belajar mengambil data dari lebih dari satu tabel. Hubungan antar tabel dibangun lewat kunci (primary key) dan kunci tamu (foreign key).

Kamus Istilah (Modul 3)

Modul join sering diuji karena menilai pemahaman relasi antar tabel.

  • RELATIONSHIP = hubungan antar tabel (dibangun dengan PK–FK).
  • JOIN = menggabungkan data dari 2+ tabel berdasarkan kolom yang berelasi.
  • INNER JOIN = hanya data yang cocok di kedua tabel.
  • LEFT JOIN = semua data kiri tampil, yang tidak cocok kanan menjadi NULL.
  • RIGHT JOIN = semua data kanan tampil, yang tidak cocok kiri menjadi NULL.
  • NULL = nilai kosong/tidak ada data (bukan 0 dan bukan string kosong).
  • UNION = menggabungkan hasil 2 query, duplikat dihapus.
  • UNION ALL = menggabungkan hasil 2 query, duplikat tetap tampil.
Materi yang sering muncul
  • INNER JOIN vs LEFT JOIN vs RIGHT JOIN.
  • NULL pada hasil OUTER JOIN.
  • UNION vs UNION ALL.
  • Alias tabel untuk merapikan query.
1) Inner Join

Inner join menampilkan data yang cocok di kedua tabel (irisan). Kalian bisa menulis join eksplisit atau implisit.

2) Outer Join

Outer join tetap menampilkan data dari salah satu sisi meski tidak ada pasangan. Sisi yang tidak cocok akan menjadi NULL.

Contoh Praktik

Contoh latihan siswa gunakan db_kuliah: mahasiswa–jurusan–fakultas dan krs–mata_kuliah–dosen.

A) Inner Join (eksplisit)
-- INNER JOIN: menggabungkan siswa + kelas (yang punya pasangan)

SELECT s.nis, s.nama, k.nama_kelas, sk.tahun_ajaran
FROM siswa s
INNER JOIN siswa_kelas sk ON s.id_siswa = sk.id_siswa
INNER JOIN kelas k ON sk.id_kelas = k.id_kelas
ORDER BY k.nama_kelas, s.nama;
B) Inner Join (implisit)
-- JOIN gaya lama (implisit) — hasilnya sama, tapi kurang rapi

SELECT s.nis, s.nama, k.nama_kelas, sk.tahun_ajaran
FROM siswa s, siswa_kelas sk, kelas k
WHERE s.id_siswa = sk.id_siswa
  AND sk.id_kelas = k.id_kelas
ORDER BY k.nama_kelas, s.nama;
C) Left Join
-- LEFT JOIN: semua siswa tetap tampil, meskipun belum pernah meminjam buku
-- Jika belum pernah meminjam, kolom peminjaman/buku akan bernilai NULL.

SELECT s.nis, s.nama, p.id_pinjam, p.status
FROM siswa s
LEFT JOIN peminjaman p ON s.id_siswa = p.id_siswa
ORDER BY s.nama;
D) Right Join
-- RIGHT JOIN: semua peminjaman tetap tampil (walau kasus siswa hilang jarang terjadi)

SELECT p.id_pinjam, s.nama, p.tanggal_pinjam, p.status
FROM siswa s
RIGHT JOIN peminjaman p ON s.id_siswa = p.id_siswa
ORDER BY p.tanggal_pinjam DESC;
E) UNION vs UNION ALL
-- UNION: menggabungkan dua daftar dengan format kolom yang sama

-- Daftar 'nama orang' di sekolah: nama siswa + wali kelas
SELECT nama AS nama_orang
FROM siswa
UNION
SELECT wali_kelas AS nama_orang
FROM kelas
ORDER BY nama_orang;
Catatan: UNION menghapus duplikat, UNION ALL menampilkan semua termasuk duplikat. Jumlah kolom dan tipe data harus sesuai.

Instruksi pengerjaan
  1. Pilih database: jalankan USE db_sekolah; (atau pilih dari phpMyAdmin).
  2. Tulis query dulu di catatan, lalu jalankan di phpMyAdmin (tab SQL).
  3. Pastikan benar: cek nama kolom, tanda petik, dan hasil output.
  4. Simpan bukti: screenshot query + tabel hasilnya.
  5. Kumpulkan: file PDF/Doc berisi jawaban (nomor + query + screenshot).
Soal latihan
Latihan 1 — INNER JOIN
  1. Tampilkan siswa dan kelasnya: siswa + siswa_kelas + kelas.
  2. Pilih kolom: nis, nama, nama_kelas, tahun_ajaran.
Latihan 2 — LEFT JOIN (NULL)
  1. Tampilkan semua siswa beserta status peminjaman (jika ada).
  2. Gunakan LEFT JOIN agar siswa tanpa pinjam tetap tampil.
  3. Tuliskan 1 kalimat: mengapa bisa muncul NULL?
Latihan 3 — UNION vs UNION ALL
  1. Buat dua SELECT sederhana yang kolomnya sama (mis. daftar kota dari siswa, daftar penulis dari buku).
  2. Gabungkan dengan UNION dan UNION ALL, bandingkan hasil duplikatnya.
Catatan guru: Jika hasil kosong, cek lagi syaratnya. Bisa jadi datanya memang tidak memenuhi kondisi. Kalian boleh menyesuaikan angka batas (misal 80, 85) asal logikanya tetap benar.
