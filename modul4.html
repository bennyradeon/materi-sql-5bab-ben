Modul 4 — Pemrosesan Form & CRUD (Create, Read, Update, Delete)

Modul ini melatih kalian menerima input dari form HTML dan memprosesnya di PHP. Kalian akan memakai metode GET (mengirim data lewat URL) dan POST (mengirim data lewat body request), lalu membaca nilainya dengan $_GET, $_POST, atau $_REQUEST.

Kamus Istilah (Modul 4)

Ini istilah web yang harus kalian hafal artinya, bukan hafal kodenya.

  • HTTP = HyperText Transfer Protocol → aturan komunikasi browser ↔ server.
  • GET = metode HTTP untuk mengirim data lewat URL (query string).
  • POST = metode HTTP untuk mengirim data lewat body request.
  • PHP = PHP: Hypertext Preprocessor → bahasa server-side untuk memproses request.
  • CRUD = Create, Read, Update, Delete → 4 operasi dasar data.
  • $_GET = variabel superglobal PHP untuk data GET.
  • $_POST = variabel superglobal PHP untuk data POST.
  • $_REQUEST = gabungan GET+POST (dipakai seperlunya).
Bedah Kode Form (HTML + PHP) — fungsi tiap bagian

Sekarang saya jelaskan bagian-bagian kode form supaya kalian paham, bukan sekadar menyalin. Perhatikan: ketika kalian membuat form, ada bagian HTML (tampil di browser) dan ada bagian PHP (diproses server).

A) Bagian HTML Form
  • <form>: wadah input. Di sinilah kalian menentukan method dan action.
  • method="get": nilai input dikirim lewat URL (terlihat di address bar).
  • method="post": nilai input dikirim lewat body request (tidak terlihat di URL).
  • action="...": menentukan file tujuan yang memproses data form.
  • <input name="nama">: name adalah “kunci” untuk mengambil nilai di PHP.
  • <input type="submit">: tombol untuk mengirim form.
B) Bagian PHP Pemrosesan
  • $_GET: tempat data form yang dikirim dengan GET.
  • $_POST: tempat data form yang dikirim dengan POST.
  • isset(...): mengecek apakah suatu data sudah ada/terisi (menghindari error).
  • echo: menampilkan output ke browser.
  • $_SERVER['PHP_SELF']: menghasilkan nama file saat ini (form diproses oleh file yang sama).
  • $_SERVER['REQUEST_METHOD']: memberi tahu metode yang dipakai (GET atau POST).
Catatan kebiasaan baik: Pastikan setiap input punya atribut name. Tanpa name, nilai tidak akan terbaca di PHP.
1) GET vs POST
  • GET: data terlihat di URL, cocok untuk pencarian.
  • POST: data tidak tampil di URL, cocok untuk form yang lebih “sensitif”.
Keduanya bisa dibaca lewat $_REQUEST.
2) Prefilling

Setelah submit, nilai input biasanya kosong lagi. Prefilling menjaga nilai tetap tampil.

Contoh Praktik
A) Form GET
<!DOCTYPE html>
<html>
<head><title>Metode GET</title></head>
<body>
<form action="<?php $_SERVER['PHP_SELF'];?>" method="get">
  Nama <input type="text" name="nama" /> <br />
  <input type="submit" value="OK" />
</form>
<?php
if (isset($_GET['nama'])) {
  echo 'Hallo, ' . $_GET['nama'];
}
?>
</body>
</html>
B) Form POST
<!DOCTYPE html>
<html>
<head><title>Metode POST</title></head>
<body>
<form action="<?php $_SERVER['PHP_SELF'];?>" method="post">
  Nama <input type="text" name="nama" /> <br />
  <input type="submit" value="OK" />
</form>
<?php
if (isset($_POST['nama'])) {
  echo 'Hallo, ' . $_POST['nama'];
}
?>
</body>
</html>
C) Identifikasi metode + $_REQUEST
<?php
if (isset($_REQUEST['nama'])) {
  echo 'Metode: ' . $_SERVER['REQUEST_METHOD'];
}
?>
Latihan (Modul 4)
Kerjakan di db_kuliah. Tulis query rapi (baris baru + indent) dan simpan hasilnya.

Instruksi pengerjaan
  1. Pilih database: jalankan USE db_kuliah; (atau pilih dari phpMyAdmin).
  2. Tulis query dulu di catatan, lalu jalankan di phpMyAdmin (tab SQL).
  3. Pastikan benar: cek nama kolom, tanda petik, dan hasil output.
  4. Simpan bukti: screenshot query + tabel hasilnya.
  5. Kumpulkan: file PDF/Doc berisi jawaban (nomor + query + screenshot).
Soal latihan
Latihan 1 — GET vs POST
  1. Buat form_get.php: input nama + submit (method GET).
  2. Buat form_post.php: input nama + submit (method POST).
  3. Uji di browser: catat perbedaan tampilan data di URL.
Latihan 2 — Prefilling
  1. Buat form POST yang setelah submit, nilai input tetap muncul (prefill).
  2. Tampilkan pesan: “Halo, [nama]”.
  3. Gunakan isset() agar tidak error saat halaman pertama dibuka.
Latihan 3 — Mini CRUD (db_kuliah)
  1. Buat form tambah data sederhana ke tabel mahasiswa (Create).
  2. Buat halaman daftar mahasiswa (Read).
  3. (Opsional) Tambahkan tombol hapus 1 data (Delete) dengan parameter id.
Pengumpulan: kirim 3 file PHP + screenshot hasil uji (URL GET, hasil POST, dan data masuk ke database bila CRUD dikerjakan).
Catatan guru: Jika hasil kosong, cek lagi syaratnya. Bisa jadi datanya memang tidak memenuhi kondisi. Kalian boleh menyesuaikan angka batas (misal 80, 85) asal logikanya tetap benar.
