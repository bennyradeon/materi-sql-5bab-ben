Modul 5 — Session, Cookies, dan Login

Modul ini mengajarkan cara membuat halaman yang “mengingat” user. Session disimpan di server, cookie disimpan di browser. Keduanya sering dipakai untuk login.

Kamus Istilah (Modul 5)

Istilah session dan cookies sering muncul sebagai konsep alur login.

  • SESSION = data sementara di server untuk user tertentu (berdasarkan session id).
  • COOKIE = data kecil di browser, bisa punya waktu kedaluwarsa (expire).
  • LOGIN = proses verifikasi user (cek username/password) lalu membuat session status.
  • LOGOUT = menghapus session agar akses kembali ditutup.
  • AUTH = Authentication → memastikan “siapa” user-nya.
  • AUTHZ = Authorization → memastikan “boleh akses apa”.
  • REDIRECT = memindahkan user ke halaman lain (header location).
  • SESSION ID = identitas sesi yang menghubungkan browser ke data session di server.
Bedah Kode Session & Cookies — fungsi tiap baris

Session dan cookies sering muncul di ujian karena kalian harus paham perannya. Saya jelaskan singkat tapi lengkap: session disimpan di server, sedangkan cookie disimpan di browser.

A) Session
  • session_start(): wajib dipanggil di awal sebelum output HTML, agar session aktif.
  • $_SESSION['key'] = value: menyimpan data session (contoh: username, status login).
  • session_unset(): menghapus semua variabel session.
  • session_destroy(): menghancurkan session (logout total).
Jika session_start() tidak dipanggil, maka $_SESSION tidak akan bekerja dengan benar.
B) Cookies
  • setcookie(name, value, expire, path): membuat cookie.
  • expire: waktu kedaluwarsa (pakai time() + detik).
  • path: biasanya '/' agar cookie berlaku di seluruh website.
  • $_COOKIE['name']: membaca nilai cookie.
  • Menghapus cookie: setcookie dengan expire waktu lampau.
Cookie bisa diubah pengguna di browser. Jadi jangan simpan data sensitif (misalnya password) di cookie.
C) Alur Login (yang harus kalian pahami)
  1. User mengisi form login (username & password).
  2. Server mengecek data ke database (SELECT).
  3. Jika benar: buat session $_SESSION['status']='login' lalu redirect ke halaman admin.
  4. Jika salah: kembali ke login dan tampilkan pesan gagal.
  5. Logout: hapus session lalu arahkan ke login.
1) Session

Mulai session dengan session_start(). Data session tersimpan untuk user tertentu.

2) Cookies

Cookie tersimpan di browser dan bisa punya waktu expired. Karena di sisi klien, cookie lebih mudah dimodifikasi.

Contoh Praktik
A) Set session
<?php
session_start();
$_SESSION['username'] = "administrator";
$_SESSION['password'] = "12345678";
?>
B) Show session
<?php
session_start();
echo "Username = " . $_SESSION['username'];
echo "<br/>";
echo "Password = " . $_SESSION['password'];
?>
C) Delete session
<?php
session_start();
session_unset();
session_destroy();
?>
D) Set cookie
<?php
setcookie('username', 'administrator', time() + (60*60*24*5), '/');
setcookie('nama', 'Budi Nurcahya', time() + (60*60*24*15), '/');
?>
Mini Proyek — Login sederhana

Tujuan: halaman admin hanya bisa diakses jika session status login aktif.

A) koneksi.php (contoh)
<?php
$host = 'localhost';
$db   = 'db_kuliah';   // gunakan database kuliah untuk praktik
$user = 'root';
$pass = '';

$conn = new mysqli($host, $user, $pass, $db);
if ($conn->connect_error) {
  die("Koneksi gagal: " . $conn->connect_error);
}
?>
B) cek_login.php (ringkas)
<?php
session_start();
include 'koneksi.php';

$username = $_POST['username'];
$password = $_POST['password'];

/*
  Catatan praktik:
  Buat tabel users di db_kuliah, contoh:
  CREATE TABLE users(
    id INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) UNIQUE,
    password VARCHAR(50)
  );
*/

$data = mysqli_query($conn, "SELECT * FROM users WHERE username='$username' AND password='$password'");
$cek  = mysqli_num_rows($data);

if ($cek > 0) {
  $_SESSION['username'] = $username;
  $_SESSION['status'] = "login";
  header("location:admin/index.php");
} else {
  header("location:login.php?pesan=gagal");
}
?>
Latihan (Modul 5)
Kerjakan di db_kuliah. Tulis query rapi (baris baru + indent) dan simpan hasilnya.

Instruksi pengerjaan
  1. Pilih database: jalankan USE db_kuliah; (atau pilih dari phpMyAdmin).
  2. Tulis query dulu di catatan, lalu jalankan di phpMyAdmin (tab SQL).
  3. Pastikan benar: cek nama kolom, tanda petik, dan hasil output.
  4. Simpan bukti: screenshot query + tabel hasilnya.
  5. Kumpulkan: file PDF/Doc berisi jawaban (nomor + query + screenshot).
Soal latihan
Latihan 1 — Session dasar
  1. Buat set_session.php untuk menyimpan $_SESSION['user'].
  2. Buat show_session.php untuk menampilkan session.
  3. Buat logout.php untuk menghapus session.
Latihan 2 — Cookies
  1. Buat set_cookie.php dan get_cookie.php.
  2. Buat delete_cookie.php (expire waktu lampau).
  3. Jelaskan 1 kalimat: bedanya cookie dan session.
Latihan 3 — Alur login sederhana (db_kuliah)
  1. Buat tabel users (username, password) dan isi 1 akun.
  2. Buat login.php + cek_login.php + admin.php.
  3. Admin hanya boleh dibuka jika session status login aktif.
Catatan: Untuk ujian, yang dinilai biasanya alur dan pemahaman fungsi session/cookie, bukan tampilan desain.
Catatan guru: Jika hasil kosong, cek lagi syaratnya. Bisa jadi datanya memang tidak memenuhi kondisi. Kalian boleh menyesuaikan angka batas (misal 80, 85) asal logikanya tetap benar.
